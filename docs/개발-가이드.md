# 사이트 개발 참고 가이드

다른 사이트를 만들 때 이 저장소의 템플릿(Nova, Aurora)에서 그대로 가져다 쓸 수 있는
**패턴·규칙·체크리스트** 모음입니다. 프레임워크 없이 순수 HTML/CSS/JS로 일관된 품질의
반응형 랜딩페이지를 빠르게 만드는 것이 목표입니다.

---

## 1. 파일 구조 규칙

한 템플릿 = 한 폴더, 항상 아래 3개 파일로 구성합니다.

```
<템플릿명>/
├── index.html   # 구조 (시맨틱 마크업)
├── styles.css   # 디자인 토큰 + 컴포넌트 + 반응형
└── script.js    # 최소한의 상호작용 (IIFE)
```

- 루트 `index.html` 은 각 템플릿으로 연결되는 **갤러리/진입 페이지**.
- 링크는 항상 **상대 경로**(`./nova/`)로 작성 → 프로젝트 페이지(`/repo/`)든 커스텀 도메인(`/`)이든 동일하게 동작.

## 2. 디자인 토큰 (`:root`)

색상·간격·반경·그림자를 `:root` 변수 한 곳에서 관리합니다. **하드코딩 금지**, 항상 변수 참조.

```css
:root {
  --bg: #0a0f1a; --surface: #131c2e; --surface-2: #182238;
  --border: #24304a; --text: #eaf0fb; --muted: #9aa8c4;
  --brand: #22d3ee; --brand-2: #6366f1; --accent: #f472b6;
  --radius: 18px; --container: 1140px;
  --grad: linear-gradient(120deg, var(--brand), var(--brand-2));
}
```

- **테마 변경**은 토큰만 교체하면 전체에 반영됨.
- **라이트/다크**는 `:root[data-theme="light"]` 블록에서 같은 토큰을 재정의(§6 참고).
- 그라디언트 텍스트: `background: var(--grad); -webkit-background-clip: text; color: transparent;`

## 3. 반응형 규칙

브레이크포인트는 **960 / 760 / 460px** 로 통일합니다.

| 폭 | 처리 |
|----|------|
| ≤ 960px | 다열 그리드 → 2열, 벤토 셀 재배치 |
| ≤ 760px | 데스크톱 내비 숨김 → **햄버거 메뉴**, 그리드 1~2열 |
| ≤ 460px | 컨테이너 패딩 축소, 모든 그리드 1열 |

- 유동 타이포: `font-size: clamp(2rem, 5vw, 3.2rem);`
- 컨테이너: `max-width: var(--container); margin-inline: auto; padding-inline: 24px;`
- 이미지: `max-width: 100%; display: block;`

## 4. 핵심 컴포넌트 패턴

- **버튼**: `.btn` 기본 + `--primary`(그라디언트) / `--ghost` / `--outline` 변형, `--lg` 크기, `--block` 풀폭. `border-radius: 999px`.
- **카드**: `background: var(--surface); border: 1px solid var(--border); border-radius: var(--radius);` + hover 시 `translateY(-3px)` & 그림자.
- **그리드**: `.grid--3`, `.grid--4` 또는 벤토(`grid-column/row: span N`).
- **스티키 내비**: `position: sticky; top: 0;` + `backdrop-filter: blur()` + 반투명 배경(`color-mix`).
- **히어로 글로우**: `radial-gradient` 여러 겹을 `filter: blur()` 처리한 장식 레이어(`aria-hidden`).

## 5. 상호작용 (script.js)

- 전체를 **IIFE**(`(function(){ "use strict"; ... })()`)로 감싸 전역 오염 방지.
- 요소 존재를 항상 확인(`if (el)`) 후 이벤트 바인딩.
- **모바일 메뉴**: `aria-expanded` 토글 + `hidden` 속성 제어, 링크 클릭 시 닫기.
- **스크롤 진입 애니메이션**: `IntersectionObserver` 로 `.is-visible` 부여, 관찰 후 `unobserve`.
- **카운트업**: `requestAnimationFrame` + easeOutCubic, `IntersectionObserver`(threshold 0.6)로 화면 진입 시 시작.

## 6. 다크/라이트 테마 토글 패턴

1. `<head>` 최상단 인라인 스크립트로 저장된 테마를 **즉시 적용**(FOUC 방지):
   ```html
   <script>(function(){try{var t=localStorage.getItem('theme')||
     (matchMedia('(prefers-color-scheme: light)').matches?'light':'dark');
     document.documentElement.setAttribute('data-theme',t);}catch(e){}})();</script>
   ```
2. CSS: `:root[data-theme="light"] { --bg:...; --text:...; }` 로 토큰 재정의.
3. JS: 버튼 클릭 시 `data-theme` 토글 + `localStorage` 저장 + `aria-pressed` 갱신.
4. 그라디언트 위 텍스트는 라이트에서 대비가 낮아질 수 있으니 별도 보정.

## 7. 접근성 체크리스트

- [ ] `lang` 속성, `<meta viewport>`, 의미 있는 `<title>`/`description`
- [ ] 시맨틱 태그(`header`/`nav`/`main`/`section`/`footer`)
- [ ] 인터랙티브 요소에 `aria-label`/`aria-expanded`/`aria-pressed`
- [ ] 장식용 요소는 `aria-hidden="true"`
- [ ] `@media (prefers-reduced-motion: reduce)` 로 애니메이션 비활성화
- [ ] 색 대비 확보(특히 그라디언트 위 텍스트), 키보드 포커스 유지

## 8. 품질 검증 (배포 전)

- HTML 태그 중첩 검증(`.github/workflows/ci.yml` 의 파서 스크립트 재사용)
- JS 문법: `node -c script.js`
- 워크플로 YAML: `python3 -c "import yaml; yaml.safe_load(open('...'))"`
- 링크·이미지 경로가 **상대 경로**인지 확인

## 9. 배포 (GitHub Pages)

1. `.github/workflows/deploy.yml` — `main` push 시 저장소 전체를 아티팩트로 업로드해 Pages 배포(빌드 단계 없음).
2. 최초 1회 **Settings → Pages → Source = GitHub Actions**.
3. 기본 주소: `https://<user>.github.io/<repo>/`

### 커스텀 도메인 연결

1. 저장소 루트에 `CNAME` 파일 생성, 내용은 도메인 한 줄(예: `templete04.dreamitbiz.com`).
2. **DNS 설정**(도메인 관리자에서): 서브도메인은 `CNAME` 레코드로
   `templete04` → `aebonlee.github.io` 등록. (루트 도메인은 A 레코드로 GitHub Pages IP 4개 지정.)
3. DNS 전파 후 Settings → Pages 에서 도메인 확인 및 **Enforce HTTPS** 체크.
4. ⚠️ DNS 설정 전 `CNAME`을 추가하면 `*.github.io` 주소가 커스텀 도메인으로 리디렉션되어
   일시적으로 접속이 안 될 수 있으니, **DNS를 먼저(또는 함께) 설정**하세요.

## 10. 새 템플릿 스캐폴딩 순서

1. `<이름>/` 폴더 + 3개 파일 생성, `:root` 토큰부터 정의.
2. 내비 → 히어로 → 본문 섹션 → CTA → 푸터 순으로 시맨틱 구조 작성.
3. 컴포넌트는 기존 클래스 네이밍(`.btn`, `.card`, `.section__head`, `.grid--N`) 재사용.
4. 960/760/460 반응형 + 접근성 체크리스트 적용.
5. 루트 `index.html` 갤러리와 `README.md` 표에 항목 추가.
6. CI 통과 확인 후 PR → 머지 → 자동 배포.
