### 10-1. Raycast Pro 히어로 배경 재작업 (붉은 유리 블롭)
- 대각선 광선이 실제 사이트와 다르다는 피드백 반영.
- raycast-pro 히어로를 실제 raycast.com처럼 **붉은/핑크/화이트가 부드럽게 번진 유리 블롭 + 그레인 노이즈**로 교체.
- skew + 강한 blur + `screen` 블렌드의 유리판(pane) 겹침, `feTurbulence` SVG 노이즈 오버레이, 밝은 블롭 위 텍스트 가독성용 text-shadow.

### 10-4. Raycast Pro 커맨드바 런처 목업 (시그니처 UI)
- 실제 raycast.com 렌더 마크업을 검토해, 아직 재현되지 않았던 **시그니처 커맨드바 런처 창**을 히어로에 추가.
- 반투명 다크 글래스 창(`backdrop-filter` blur+saturate) — 검색 입력(로고·플레이스홀더·깜빡이는 캐럿·AI 힌트), Favorites/Suggestions 섹션, 앱 아이콘 + 커맨드명 + 타입 라벨, 활성 행 하이라이트, 하단 **Actions ⌘K** 푸터(Raycast 브랜드·Open Command ↵).
- `script.js`에 이미 있던 `.rc__row` hover 로직(활성 행 이동)을 실제로 활성화 — 마크업이 없어 죽어 있던 코드를 되살림.
- 등장 애니메이션·`prefers-reduced-motion` 정지·모바일 축약(좁은 화면은 Actions만) 대응. 의존성 0.

### 10-3. Raycast Pro 히어로 WebGL 배경
- 실제 raycast.com 히어로가 **Three.js WebGL 유리 큐브**(canvas r167)임을 확인.
- 라이브러리 없이 **순수 WebGL 프래그먼트 셰이더**로 살아 움직이는 붉은 유리 블롭 구현 — domain-warped fbm 노이즈 + 실제 큐브 셰이더 색상(near-white/pink/red/deep) + 그레인.
- `prefers-reduced-motion`·WebGL 미지원·탭 비활성 대응(정지/폴백), 실패 시 정적 CSS 블롭으로 폴백. 의존성 0.
